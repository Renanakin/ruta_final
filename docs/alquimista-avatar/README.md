# Proyecto Alquimista Avatar

Plan maestro de ejecucion por fases para integrar la nueva identidad visual y narrativa del Alquimista en Ruta del Nido.

Este documento existe para ordenar el trabajo antes de tocar la web y para asegurar que cada fase tenga entregables, revision y aprobacion explicita.

## Regla principal

No se modifica la web hasta que el usuario apruebe la propuesta y habilite la ejecucion con una confirmacion explicita.

Gate obligatorio:

- Si el usuario no dice `VAMOS`, no se implementa ninguna fase de frontend.

## Objetivo del proyecto

Convertir al Alquimista en una identidad viva y reconocible dentro de la marca, usando el nuevo avatar como nucleo visual, narrativo y funcional de la experiencia.

El cambio debe:

- reforzar la personalidad del Alquimista
- explicar claramente como se usa
- mejorar la percepcion premium y artesanal
- elevar el valor de la interaccion IA
- mantener coherencia con Ruta del Nido

## Estructura funcional aprobada por el usuario

La web del Alquimista debe organizarse en:

1. Hero: avatar + presentacion del Alquimista + nueva atmosfera visual.
2. Seccion 2: adaptacion de la idea actual a la nueva identidad del avatar.
3. Seccion 3: mantener estructura, adaptada al nuevo personaje.
4. Seccion 4: modulo central de interaccion IA, reemplazando el logo de Ruta del Nido por el avatar del Alquimista y volviendolo la seccion mas impactante.
5. Seccion 5: adaptacion al nuevo universo visual.
6. Footer: se mantiene, usando el avatar solo si mejora el cierre.

Ademas, debe aparecer claramente el flujo de uso:

1. Comprar productos de la Ruta del Nido.
2. Ingresar el codigo en la seccion dedicada.
3. Describir los ingredientes que tiene el usuario para generar la receta.

## Fases oficiales

## Fase A0 - Alineacion y bloqueo de alcance

Objetivo:
Dejar fijado el alcance artistico, tecnico y narrativo antes de editar componentes.

Entregables:

- propuesta tecnica y artistica aprobada
- estructura final de secciones validada
- redaccion del flujo de uso validada
- criterio de uso del avatar definido

Criterio de pase:

- el usuario aprueba la direccion creativa
- el usuario autoriza la ejecucion

Estado:

- [ ] A0 aprobado

## Fase A1 - Sistema visual del avatar

Objetivo:
Definir como vive el avatar en la experiencia.

Alcance:

- recorte y ubicacion del asset
- paleta derivada del avatar
- glow, halos y acentos
- comportamiento motion base
- reglas de uso del personaje

Entregables:

- hero art direction
- reglas de escala y posicion del avatar
- propuesta de color y textura
- motion thesis de la experiencia

Criterio de pase:

- el avatar se ve integrado a la marca
- no se siente pegado ni decorativo

Estado:

- [ ] A1 aprobado

## Fase A2 - Narrativa y story system

Objetivo:
Construir la voz y la historia del Alquimista.

Alcance:

- story book del personaje
- copy del hero
- copy de secciones 2, 3 y 5
- bloque explicativo de como interactuar con el Alquimista

Entregables:

- history book
- copy maestro por seccion
- microcopys de CTA y onboarding

Criterio de pase:

- el personaje se entiende
- el flujo de uso queda claro en segundos

Estado:

- [ ] A2 aprobado

## Fase A3 - Integracion por secciones

Objetivo:
Aplicar el rediseño del avatar a toda la web del Alquimista.

Alcance:

- hero
- seccion 2
- seccion 3
- seccion 4 de interaccion
- seccion 5
- cierre y footer si aplica

Entregables:

- nueva composicion visual por seccion
- layout responsive
- integracion del avatar en desktop y mobile

Criterio de pase:

- consistencia visual entre secciones
- identidad clara del personaje en toda la experiencia

Estado:

- [ ] A3 aprobado

## Fase A4 - Seccion de interaccion IA

Objetivo:
Volver la seccion de IA la pieza mas memorable y mas clara del recorrido.

Alcance:

- reemplazo de logo por avatar
- panel central de interaccion
- estados de input, espera, resultado y fallback
- explicacion del ritual de uso
- tono premium y util

Entregables:

- modulo de interaccion rediseñado
- CTA y entradas mas claras
- fallback a WhatsApp alineado

Criterio de pase:

- la interaccion se entiende rapido
- el modulo se siente premium, claro y emocionante

Estado:

- [ ] A4 aprobado

## Fase A5 - QA y cierre

Objetivo:
Validar calidad visual, UX, responsive y consistencia narrativa.

Entregables:

- QA visual desktop y mobile
- validacion responsive
- verificacion de accesibilidad basica
- documento de cierre

Criterio de pase:

- experiencia estable
- sin quiebres visuales
- sin contradicciones de copy

Estado:

- [ ] A5 aprobado

## Evidencia requerida por fase

Cada fase debe cerrar con:

- archivos tocados o propuestos
- resumen de cambios
- riesgos residuales
- evidencia visual o tecnica

## Que no se puede hacer

- cambiar la web sin aprobacion del usuario
- reinterpretar la identidad del avatar sin validacion
- usar el avatar como decoracion sin funcion
- convertir al Alquimista en chatbot generico
- romper el tono premium y artesanal de Ruta del Nido

## Resultado esperado

Una web del Alquimista mas narrativa, clara y memorable, donde el avatar actua como identidad central, guia de uso y sello emocional de la experiencia.
