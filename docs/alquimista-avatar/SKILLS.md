# Skills y roles para Proyecto Alquimista Avatar

Matriz operativa para ejecutar el proyecto por fases sin improvisacion.

## Regla de ejecucion

Hasta que el usuario apruebe y diga `VAMOS`, esta matriz sirve solo para planificacion, no para implementacion.

## Roles principales

### Product Manager

Responsabilidad:

- traducir el requerimiento en fases claras
- priorizar decisiones de impacto
- proteger coherencia entre narrativa, UX y conversion

Preguntas que responde:

- que se implementa primero
- que se aprueba antes de tocar codigo
- como se mide exito por fase

Rol recomendado:

- `planner`
- apoyo de `writer`

## Diseñador de imagen / identidad visual

Responsabilidad:

- integrar el avatar como pieza central del sistema visual
- definir composicion, escala, recorte, glow y atmosfera
- asegurar coherencia con la marca Ruta del Nido

Preguntas que responde:

- donde vive el avatar en cada seccion
- como dialoga con el fondo y la tipografia
- como mantenerlo premium y no infantil

Skills / roles recomendados:

- `designer`
- `vision`
- `build-web-apps:frontend-skill`

## UX Expert

Responsabilidad:

- garantizar comprension inmediata del flujo de uso
- asegurar que el personaje aporte claridad y no ruido
- ordenar jerarquia, CTAs y lectura

Preguntas que responde:

- se entiende quien es el Alquimista
- se entiende como interactuar con el modulo
- se entiende que hacer primero, despues y al final

Skills / roles recomendados:

- `designer`
- `critic`
- `build-web-apps:web-design-guidelines`

## Animador experto en UX

Responsabilidad:

- definir motion funcional y atmosferico
- crear entradas, respiracion visual y transiciones utiles
- evitar ruido, mareo o exceso teatral

Preguntas que responde:

- que anima y que queda quieto
- como el avatar cobra vida sin distraer
- como se refuerzan estados del sistema

Skills / roles recomendados:

- `animation-designer`
- `designer`

## Writer / narrativa de personaje

Responsabilidad:

- escribir history book
- definir voz del Alquimista
- redactar onboarding y ritual de uso

Preguntas que responde:

- como habla el personaje
- como se cuenta su origen
- como se explica el flujo de uso con tono premium y claro

Roles recomendados:

- `writer`
- `planner`

## Criterios de uso del avatar

- siempre debe cumplir una funcion narrativa o UX
- nunca debe sentirse decorativo sin proposito
- debe ser reconocible en hero, interaccion y cierre
- debe integrarse al tono premium, artesanal y cercano

## Criterios de rechazo

Una fase debe devolverse si:

- el avatar parece un sticker sobre la interfaz
- el flujo de uso no se entiende en menos de 10 segundos
- el modulo IA pierde claridad
- la experiencia se vuelve infantil o demasiado generica
- el motion distrae mas de lo que ayuda

## Gate final

No se pasa de planificacion a implementacion hasta que el usuario revise estos documentos y apruebe explicitamente la ejecucion.
